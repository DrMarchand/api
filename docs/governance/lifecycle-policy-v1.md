# Lifecycle Policy v1

Date Established: 2026-06-15
Corrected: 2026-06-15
Originating Author: Joseph Kyle Marchand
Business Root: Design Orchard LLC

## Authority

Source of truth.

May create, modify, and own records.

## Active

Current operational work.

Expected to change.

## Boiler Room

Documents layer.

Working files that are weird, unfinished, legacy, experimental, or poorly understood, but still do something.

These belong in Documents or a controlled document authority until understood.

Rule:

Do not delete until purpose is documented.

Examples:

- working documents
- strange but useful scripts
- old but functional records
- bridge notes
- migration instructions
- working reference files
- weird files that still do something

## Dumpster

Downloads layer.

Downloads is the intake and staging area.

Most files in Downloads should eventually go to:

- Documents
- Creative Canvas
- Library
- GitHub
- Trash

Rule:

Downloads should not be treated as authority.

Examples:

- downloaded files
- temporary exports
- installer files
- imported camera uploads
- staging zips
- pending review material

## Graveyard

No Man's Land.

Files that once existed, were emptied from Trash, are corrupted, are disconnected, or only remain as evidence that something used to exist.

Graveyard is not normal storage.

It is a memory of loss, corruption, or deletion.

Examples:

- corrupted files
- broken aliases
- broken symlinks
- dead shortcuts
- deleted-file records
- missing targets
- emptied-trash evidence
- ghost paths
- unrecoverable references

## Ashes

Fully removed after documentation, evidence, and approval.

Ashes means the object is no longer recoverable through normal system records.

## Toolbox

Blueprint layer.

The Toolbox holds plans, templates, schemas, diagrams, build instructions, system maps, and architecture documents.

Toolbox files are not trash, not ghosts, and not random working documents.

They are reusable construction knowledge.
